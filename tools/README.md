# LoraLeya — Tools

Одноразовые скрипты для настройки сайта. Запускаются по URL под учёткой администратора.
После выполнения всех задач файлы можно удалить (история сохранится в git).

---

## import-products.php

**Что делает:** создаёт товары WooCommerce (5 видов) для одного цвета из жёстко
заданного каталога в коде. Для каждого цвета — вариативные товары с правильными
вариациями, ценами, SKU, привязкой к категориям и таксономии `pa_fabric_color`.

**Как запустить:**
```
https://loraleya.ru/wp-content/themes/loraleya/tools/import-products.php?color=biruza
```
Параметр `color` — slug термина из таксономии `pa_fabric_color`.

**Формат вывода:** HTML-страница с построчным логом по фазам (категории → товары →
вариации) и итоговой таблицей со ссылками в wp-admin.

**Идемпотентен:** товар с таким же slug не создаётся повторно, вариации обновляются.

**После успеха:** запустить для других цветов (`?color=bezheviy`, `?color=beliy`, ...).
Файл можно удалить после импорта всех 17 цветов.

## import-photos.php

**Что делает:** массово загружает файлы из временной папки
`wp-content/uploads/_import-photos/{color}/` в WordPress Media Library.

**Как запустить:**
```
https://loraleya.ru/wp-content/themes/loraleya/tools/import-photos.php
```
Без параметров — обрабатывает все подпапки цветов автоматически.

**Формат вывода:** HTML-страница с построчным логом по подпапкам (цветам).
```
✅ beliy-kuvert.webp → загружен (post_id 156)
⏭  biruza-dorozhka.webp → пропущен (папка biruza исключена)
⚠  goluboy-kuvert.webp → пропущен (уже существует, post_id 87)
❌ blek-zoloto-macro.webp → ошибка: <текст>
```

**Правила обработки:**
- Папка `biruza/` — пропускается полностью (загружена вручную)
- Файл уже в Media Library (по `post_title`) — пропускается (идемпотентность)
- Префикс файла не совпадает с именем папки — пропускается с предупреждением
- Файл новый → копируется в `uploads/YYYY/MM/`, создаётся запись в Media Library

**После успеха:** папку `_import-photos/` удаляет заказчик через Диспетчер файлов reg.ru.
Скрипт папку не трогает. Файл скрипта можно удалить.
